# API Reference — `nebula-core`

Public surface based on `crates/core/src/*`. All items are re-exported from the crate root via wildcard re-exports.

## Quick Start

```rust
use nebula_core::prelude::*;

let exec_id = ExecutionId::new();
let node_id  = NodeId::new();
let scope    = ScopeLevel::Action(exec_id, node_id);
assert!(scope.is_action());

let key: PluginKey = "HTTP Request".parse().unwrap();
assert_eq!(key.as_str(), "http_request");

let err = CoreError::not_found("workflow", "abc-123");
assert!(err.is_client_error());
assert!(!err.is_retryable());
```

---

## Public Surface

- **Stable APIs:** All re-exported types, traits, constants. No deprecations.
- **Experimental APIs:** None. Core is conservative.
- **Internal:** `collapse_underscores` (private), `domain_key` macro internals.

## IDs (`id.rs`)

Eleven strongly typed UUID wrappers generated by `domain_key::define_uuid!`. Each is a distinct newtype: passing `NodeId` where `WorkflowId` is expected is a compile error.

**ID types:**

| Type | Domain string |
|------|---------------|
| `UserId` | `"user"` |
| `TenantId` | `"tenant"` |
| `ExecutionId` | `"execution"` |
| `WorkflowId` | `"workflow"` |
| `NodeId` | `"node"` |
| `ActionId` | `"action"` |
| `ResourceId` | `"resource"` |
| `CredentialId` | `"credential"` |
| `ProjectId` | `"project"` |
| `RoleId` | `"role"` |
| `OrganizationId` | `"organization"` |

**Properties:** `Copy`, `Clone`, `Debug`, `PartialEq`, `Eq`, `Hash`, `PartialOrd`, `Ord`, `Serialize`/`Deserialize` (as UUID string), `Display` (UUID string), `FromStr`.

**Methods (same for all IDs):**

| Method | Signature | Notes |
|--------|-----------|-------|
| `new` | `fn new() -> Self` | Generate random UUID |
| `nil` | `fn nil() -> Self` | All-zero sentinel UUID |
| `from` | `impl From<uuid::Uuid> for Id` | Wrap an existing UUID |
| `get` | `fn get(&self) -> uuid::Uuid` | Unwrap the inner UUID |
| `parse` | `fn parse(s: &str) -> Result<Self, UuidParseError>` | Parse from UUID string |
| `from_bytes` | `fn from_bytes([u8; 16]) -> Self` | Construct from raw bytes |
| `as_bytes` | `fn as_bytes(&self) -> &[u8; 16]` | Borrow raw bytes |
| `is_nil` | `fn is_nil(&self) -> bool` | True if all-zero |
| `domain` | `fn domain(&self) -> &str` | Returns the domain string |

**Conversions:**

| Trait | Implementation |
|-------|---------------|
| `From<uuid::Uuid>` | Wrap existing UUID |
| `TryFrom<&str>` | Delegates to `parse` |
| `TryFrom<String>` | Delegates to `parse` |
| `From<Self> for String` | UUID string |

```rust

assert_eq!(id, parsed);
assert_eq!(id.domain(), "execution");
```

## Scoping (`scope.rs`)

### `ScopeLevel`

```rust
pub enum ScopeLevel {
    Global,
    Organization(OrganizationId),
    Project(ProjectId),
    Workflow(WorkflowId),
    Execution(ExecutionId),
    Action(ExecutionId, NodeId),
}
```

**Classification predicates:**

| Method | True when |
|--------|-----------|
| `is_global()` | `Global` |
| `is_organization()` | `Organization(_)` |
| `is_project()` | `Project(_)` |
| `is_workflow()` | `Workflow(_)` |
| `is_execution()` | `Execution(_)` |
| `is_action()` | `Action(_, _)` |

**ID accessors:**

| Method | Returns | Available for |
|--------|---------|---------------|
| `organization_id()` | `Option<&OrganizationId>` | `Organization` only |
| `project_id()` | `Option<&ProjectId>` | `Project` only |
| `workflow_id()` | `Option<&WorkflowId>` | `Workflow` only — **not** `Execution` or `Action` |
| `execution_id()` | `Option<&ExecutionId>` | `Execution` and `Action` |
| `node_id()` | `Option<&NodeId>` | `Action` only |

> **Caveat:** `workflow_id()` returns `None` for `Execution` and `Action` variants even though they imply a workflow context. Use `HasContext` on concrete types to carry workflow association.

**Containment:**

```rust
fn is_contained_in(&self, scope: &ScopeLevel) -> bool
```

Rules:
- Everything is contained in `Global`.
- A scope is contained in itself (reflexive).
- Deeper scopes (Organization, Project, Workflow, Execution, Action) are contained in `Global`.
- For `Execution`→`Workflow` and `Action`→`Workflow`/`Execution` containment: checks scope level only, **does not verify IDs match**.

**Parent/child:**

```rust
fn parent(&self) -> Option<ScopeLevel>
fn child(&self, child_type: ChildScopeType) -> Option<ScopeLevel>
```

`parent()` returns `Some` only for:
- `Organization(id)` → `Some(Global)`
- `Action(exec, node)` → `Some(Execution(exec))`

All other variants return `None` (parent IDs are not carried in scope).

### `ChildScopeType`

```rust
pub enum ChildScopeType {
    Organization,
    Project,
    Workflow,
    Execution,
    Action,
}
```

Used as argument to `ScopeLevel::child()`.

### `ScopedId`

Associates an arbitrary string key with a scope:

```rust

```

**Constructors:**

| Constructor | Signature |
|-------------|-----------|
| `global` | `fn global(id: impl Into<String>) -> Self` |
| `workflow` | `fn workflow(workflow_id: WorkflowId, id: impl Into<String>) -> Self` |

## Keys (`keys.rs`)

### `PluginKey`

Normalized, validated plugin type identifier.

**Normalization pipeline (applied in order):**
1. Trim leading/trailing whitespace
2. Lowercase
3. Replace whitespace and `-` with `_`
4. Collapse consecutive `_` to single `_`
5. Strip leading/trailing `_`

**Constraints after normalization:**
- Non-empty
- Characters: `[a-z_]` only
- Maximum 64 characters

```rust
// Creation
PluginKey::new(raw: &str) -> Result<Self, PluginKeyError>
str::parse::<PluginKey>()           // via FromStr
PluginKey::try_from(&str / String)  // via TryFrom

// Access
key.as_str() -> &str
key.to_string() -> String           // via Display
String::from(key)

// Equality
key == "slack"      // PartialEq<str>
key == &"slack"     // PartialEq<&str>
key == "slack".to_string()  // PartialEq<String>
```

**`PluginKeyError` variants:**

| Variant | When |
|---------|------|
| `Empty` | Input is empty, whitespace-only, or underscore-only |
| `InvalidCharacters` | Contains chars outside `[a-z_]` after normalization |
| `TooLong` | Normalized key > 64 chars |

Serde: normalizes on deserialize; serializes to plain string. Rejects invalid JSON strings.

### `ParameterKey` and `CredentialKey`

Generated by `domain_key::key_type!` macro with domains `"parameter"` and `"credential"` respectively. They carry domain-validated string keys. `KeyParseError` is re-exported for error handling.

## Error Semantics

- **Retryable errors:** `Timeout`, `RateLimitExceeded`, `ServiceUnavailable`, `Storage`, `Network { retryable: true }`, `NodeExecution { retryable: true }`. Use `is_retryable()` — do not match variants directly.
- **Fatal (non-retryable):** `Validation`, `NotFound`, `AlreadyExists`, `PermissionDenied`, `Authentication`, `Authorization`, `InvalidInput`, `InvalidState`, `Internal`, `Configuration`, `Cluster`, `Serialization`, domain errors.
- **`error_code()`:** stable ASCII string, suitable for logs/metrics. See table above.
- **`user_message()`:** safe for display to end users; omits internal details.

## Usage Patterns

- **ID creation:** `Id::new()` for new entities; `Id::parse(s)` for deserialization; `Id::nil()` for sentinel/default.
- **Scope construction:** `ScopeLevel::Action(exec_id, node_id)` for action-scoped resources; `ScopedId::execution(exec_id, "key")` for execution-scoped keys.
- **Context propagation:** Build `OperationContext` with builder pattern; pass by reference.
- **Error handling:** Use typed constructors (`CoreError::validation()`, etc.); check `is_retryable()` for retry decisions.
- **Plugin keys:** Always parse/construct via `PluginKey::new()` or `.parse()`; never store raw strings.

## Compatibility Rules

- **Major version bump:** New `CoreError` variants that consumers must match; variant renames; serde schema changes; ID/scope semantic changes.
- **Deprecation policy:** Minimum 6 months; documented in MIGRATION.md; `#[deprecated]` with replacement path.
