# nebula-resource — Synthesis Report

Paths relative to `crates/resource/src/` unless prefixed. Crate ~17.7k LoC (src + macros), `#![forbid(unsafe_code)]`, RPITIT throughout (no async_trait).

## A. ARCHITECTURE MAP

nebula-resource is the engine-owned "bulkhead pool" (canon L2-§11.4/§13.3): resources are authored as a typed `Resource` trait impl with credential slots (`SlotCell<CredentialGuard<C>>`) populated by the framework, registered through a single `Manager::register(RegistrationSpec)` funnel into a multi-tenant `Registry` keyed by `(ResourceKey, scope, SlotIdentity)`, acquired through a shared TOCTOU-hardened pipeline that hands out drop-releasing `ResourceGuard`s whose teardown rides a best-effort `ReleaseQueue`, and rotated/revoked via a two-phase (sync taint + epoch bump, then drain + hook) protocol fanned out from the engine's credential event bus. Three topologies (Pooled / Resident / Bounded-typestate) own instance lifecycle; the only production consumer is the engine, which uses exactly two entry points (`acquire_erased_for`, `register_resolved`) plus the rotation pair — and no `Manager` is constructed in `apps/server` at all, so the entire subsystem is currently dark in production pending the M12.4 bind-population producer.

**10 most load-bearing types/flows:**

1. `Resource` trait — `resource.rs:249-489` (4 assoc types `Config/Runtime/Lease/Error`, `create` :292, rotation hooks `on_credential_refresh` :320 / `on_credential_revoke` :350, `credential_slot_epoch` :400).
2. `SlotCell<S>` — `slot.rs:57-230` (`ArcSwapOption<SlotEntry{generation,value}>`, torn-read-free `load_versioned` :166, writer mutex, lock-free readers) — the credential↔resource seam.
3. `ResourceGuard<R>` + `GuardInner{Owned,Guarded,Shared}` — `guard.rs:111-169`; sync `Drop` → ReleaseQueue `guard.rs:596-718`; awaited `release()` via detached spawn `guard.rs:531-570`; dual drain trackers `guard.rs:59-65`.

## B. VERIFIED WARTS

Deduplicated; multi-reader confirmations noted. No unresolved contradictions found (the two near-conflicts: README-vs-ROADMAP on fan-out resolves in favor of ROADMAP — fan-out IS shipped, README is stale; `get_any` multi-tenant "inactive" is fail-closed by design, re-framed as operator-surface gap, not a leak).

### Contract-breaking / HIGH

**B1. `fingerprint()` default makes hot-reload a silent no-op and its doc is inverted.** (3 readers) `resource.rs:64-70` doc: "default returns 0 (always different)"; `manager/registration.rs:515-519` does `if new_fp == old_fp → NoChange` — two default configs fingerprint 0, compare **equal**, new config silently discarded. The example propagates the misreading (`examples/examples/resource_resident_http.rs:202-206`). Fix: ship `#[derive(ResourceConfig)]` with structural-hash fingerprint as the default; make the no-override case a compile-or-registration error rather than a silent equal-zero; fix the doc.

**B2. Hot reload never touches live runtimes; `ReloadOutcome` is a dishonest type.** (4 readers; #712 acknowledged) `registration.rs:546-554` always returns `SwappedImmediately`; pool topology Config (max_size/TTLs/strategy) immutable post-registration (`registration.rs:161-171`), Bounded config swap is a no-op, Resident converges only via recreate (`reload.rs:18-23`). Zero production callers of `reload_config`. Fix: drain-then-rebuild per topology with truthful per-topology outcome variants; adopt sqlx `set_connect_options` + deadpool `resize`/`retain` seams (research 1 §3, §1).

**B3. `ErrorKind` retry taxonomy destroyed at the only boundary actions use.** (3 readers) The crate builds careful classification (`error.rs:15-85`, 8 variants, retry hints :157-164) then flattens **every** acquire failure to `ErrorKind::Permanent`: `ext.rs:127-129, 151-169`, `resource_ref.rs:111-113`; action side maps to `ActionError::fatal` (`crates/action/src/context.rs:755-766`); `EngineResourceAccessor` special-cases only NotFound (`resource_accessor.rs:131`). Transient/Exhausted{retry_after}/Backpressure never reach a workflow retry policy. The exact bb8-class failure research 1 warns about (fail-fast vs retry must be typed through the boundary). Fix: thread `ErrorKind` (or `nebula_error::Classify`) through the erased seam; dedup the two copied downcast/flatten bodies (`ext.rs:122-171` ≡ `resource_ref.rs:97-129`).

**B4. `RegistrationSpec.topology` ↔ `acquire: ErasedAcquireFn` redundancy: mismatch compiles, fails at acquire time.** (5 readers) Every call site hand-pairs `TopologyRuntime::Pool(...)` with `Manager::erased_acquire_pooled_for::<R>()` (`manager/options.rs:233-250`; example `examples/examples/resource_postgres_pool.rs:372-380`; 20+ times in tests); mismatch surfaces only via `unexpected_topology` (`acquire.rs:407`). The hook is fully derivable from the topology variant. Fix: typed `register_pooled<R: Pooled>`-class constructors (or spec constructor deriving the fn from the variant) — the crate's own Bounded typestate proves it knows how (`tests/derive_resource_compile_fail.rs:52-61`). Also restore an ergonomic tier: the funnel consolidation deleted ~17 shorthands and left ~75 lines of ceremony for a minimal resource (measured, reader 0).

**B5. `#[derive(Resource)]` emits `create() { todo!() }` — coherence forbids the user supplying `create`, making the flagship derive unusable.** (2 readers) `macros/src/resource.rs:79-90`; macro doc claims the implementor supplies `create` (`macros/src/lib.rs:33-35`) — impossible. Proven in-repo: every real resource is hand-written; hand-written impls must then hand-mirror the epoch fold (`tests/resident_rotation_race.rs:180-187`), reintroducing the discipline hazard the derive exists to remove; hand impls also silently get `credential_slot_epoch() == 0`, disabling staleness detection with no lint (`resource.rs:390-393`). Fix: split derives — a standalone slots/`DeclaresDependencies` derive that composes with a hand-written `impl Resource` — or a `create = path` escape hatch. Never emit a panicking body.

**B6. `remove()` / silent re-register are multi-tenant nukes with no teardown.** (3 readers) `registry.rs:745-751` drops the whole `Vec` — every scope and tenant row for a key; `Manager::remove` (`registration.rs:566-577`) adds no drain, no async `destroy()` of idle pooled instances (sync Arc drop leaks server-side handles, #713-class), and records `record_destroy()` for a deregistration. `register()` same-triple replace (`registration.rs:50-51`) has the identical hole. No per-row deregistration exists at all. Fix: per-row `deregister(key, scope, identity)` with drain+destroy semantics, reusing the revoke drain machinery.

**B7. Pool deadline contract silently broken; Resident/Bounded ignore deadlines.** (reader 2, corroborated by reader 1's no-pipeline-timeout) Permit wait budget = `options.remaining().unwrap_or(create_timeout)` (`runtime/pool.rs:675`) — `create_timeout` overloaded as wait timeout; `create_entry` then starts a **fresh** absolute deadline ignoring the caller's (`pool.rs:716, 734, 754`) — worst case 2× create_timeout. Resident discards `AcquireOptions` wholesale (`runtime/resident.rs:243`); Bounded's `acquire_one` is unbounded after the permit (`runtime/bounded.rs:357-360`). And the only production caller never sets a deadline anyway (`resource_accessor.rs:111,127`). Fix: deadpool-style `Timeouts{wait, create, recycle}` (research 1 §1), thread one absolute deadline end-to-end, make timeout errors carry the failing phase (sqlx's #1 recurring complaint — opaque `PoolTimedOut`).

**B8. Rotation head-of-line blocking: idle Mutex held across every per-entry hook await blocks ALL pool acquires.** (reader 2) `runtime/pool.rs:336-360`; doc claims new-instance creation unaffected (:321-323) but every acquire first parks on the idle Mutex in `try_acquire_idle` (:581). Per-entry hook also unbounded (admitted :324-328). Fix: snapshot entries out of the lock, or generation-fence and dispatch hooks lock-free.

**B9. Massive dead exported public surface — semver liability before 1.0.** (3 readers, grep-verified) Zero consumers outside crate tests/examples: `RegisterOptions` + `effective_slot_identity` (`manager/options.rs:139-207`; doc references deleted `register_*_with` methods :119; acquire docs link the dead type `acquire.rs:309,534`); `DedupKey` (`dedup.rs:113-132`); `GuardInner::Shared`/`ResourceGuard::shared` (`guard.rs:248-275`, no producer, carried through 9 match arms); `RecoveryWaiter` (discarded at `manager/gate.rs:103`); the **entire Bounded topology** (~900 LoC, no consumer or example); `RecoveryGate` family (engine threads `Option<Arc<RecoveryGate>>`, every site passes `None`); `Resident::stale_after` (`topology/resident.rs:35-37` — promised in `docs/topology-reference.md:128,166`, never read); `BoundedConfig::drain_timeout` (`topology/bounded.rs:344`, never read); `reload_config`/`ReloadOutcome`; `warmup_pool`, `health_check`, `subscribe_events` (15 `ResourceEvent` variants, no production subscriber), `ResourceOpsMetrics` (never enabled outside tests); `HasResourcesExt`/`ResourceRef::resolve` (production uses `acquire_resource_by_id`); `Registry`/`AnyManagedResource`/`ReleaseQueue` re-exported (`lib.rs:107-108`) despite self-described engine-internal. Fix: aggressive `pub(crate)` demotion + delete-or-wire decision per item before any API freeze.

### Correctness / resilience — MEDIUM

**B10. `RecoveryGate` bricks in ~30s and there is no unbrick surface.** (readers 1, 4) Default `max_attempts=5, base=1s` (`recovery/gate.rs:93-100`) → backend down ~30s ⇒ `PermanentlyFailed` (:484-507); only `gate.reset()` recovers, Manager exposes no reset, no operator/API surface documented anywhere (`docs/recovery.md:92`). Cancelled probe burns an attempt: `RecoveryTicket::Drop` auto-fails (:189-206), so 5 caller cancellations during a Failed window = permanent brick (#716 acknowledges panic-as-attempt only). Fix: half-open auto-recovery, distinguish cancel from failure in ticket Drop, Manager-level reset, align backoff with `nebula_resilience::BackoffConfig` (+jitter; #718 sibling).

**B11. Taint is invisible to health.** (reader 1) `taint_now` (`rotation.rs:245-267`) sets only the AtomicBool — no phase transition, no event until drain tail; `ResourceHealthSnapshot` has no `tainted` field (`mod.rs:372-384`); `health_check` reports Ready while every acquire returns Revoked. Snapshot also embeds manager-wide counters labeled per-key (`mod.rs:708`). Fix: taint → phase/event + snapshot field.

**B12. ReleaseQueue accounting and flood holes.** (reader 1) Timed-out task (30s) dropped with warn only — `dropped_count` not incremented, falsifying the documented "non-zero dropped_count = leak warrant" invariant (`release_queue.rs:349-360` vs :285-290). `spawn_rescue` = one detached task per saturated submit, unbounded count (:240-278). All tuning hardcoded (256/4096/30s/30s, `release_queue.rs:41-60`; event bus 256 `mod.rs:532`) — `ManagerConfig` exposes only worker count. Worker `JoinError`s discarded (:316-319).

**B13. Two divergent teardown executions, double boxing, runtime-panic requirement.** (readers 0, 5) `release()` spawns a detached task per call (`guard.rs:541` — panics outside a Tokio runtime, undocumented, in a no-panic crate); `Drop` submits to the worker queue (`guard.rs:655-663`) — same teardown, two scheduling paths with different observability. Drop path double-boxes the future (`guard.rs:656-663, 684-691`). `taint()` silently no-ops for Owned guards (`guard.rs:307-314`) — and Resident acquires hand out Owned guards (`guard.rs:87-90, 148-151`), so tainting a misbehaving resident lease does nothing, silently.

**B14. Registry: two scope-resolution policies + unconstrained pub register.** (reader 0) `get`/`get_typed`/`lookup` probe exact→Global only (`registry.rs:864-877`) while acquire walks the full scope ladder (:551, :609) — `lookup`/`reload_config` cannot see a Workspace row that `acquire` would resolve. 8 public/crate lookup entry points over 5 finders across 3 orthogonal axes. `Registry::register` accepts an unconstrained `(key, type_id, managed)` triple (:407-415) — mismatch silently corrupts `type_index`; consistency only by Manager-funnel discipline. Linear `Vec` scan repeats per scope level under the DashMap shard guard (:551-577, 609-626). Fix: one resolution policy, constrain register's triple structurally, fold the lookup matrix.

**B15. Pool instance-duplication on checkout.** (reader 2) `build_guarded_handle` moves the full `PoolEntry` incl. a `runtime` clone into the release callback (`pool.rs:838-863`; lease minted via `entry.runtime.clone().into()` :552) — only metrics/fingerprint/epoch are needed; nothing in `Pooled`'s contract says Runtime⇄Lease round-trips the same instance, so a value-semantic Runtime keeps two live instances per checkout and recycles a possibly-different one. Related: `Pooled`'s bidirectional `Runtime: Into<Lease>, Lease: Into<Runtime>` bounds (`topology/pooled.rs:69-75`) make the 4th assoc type pure ceremony on the pooled path.

**B16. `Pooled::prepare` doc contradicts impl.** (reader 2) Doc: "destroy and try another one" (`topology/pooled.rs:104-105`); reality: destroy + **return Err to caller** (`pool.rs:645-649`, also :541-546). No try-next loop exists.

**B17. Resident recreate path leaks.** (reader 2) Hardcoded 10s destroy timeout, result `let _ =` discarded, no log (`resident.rs:274-277`); `Arc::try_unwrap` failure drops the old runtime without `destroy` (:278-287).

**B18. Bounded/Exclusive semantics.** (reader 2) S4 destroy on failed reset destroys `(*runtime).clone()` (`bounded.rs:500-519`) — destroys shared server-side state (Arc runtimes) or a dead copy (value runtimes); poison is permanent, no unpoison/rebuild primitive, no `ResourceEvent` emitted. Keepalive: lazily started on first acquire (the unprotected window is exactly pre-first-acquire, :251-281, 319), doc falsely claims it only runs when overridden (:85-89), keeps ticking after poison. `BoundedRuntime::new` takes and discards `resource: &R` (:230). `Capped<N>` hard-codes the cap at compile time — an operator-configurable `max_sessions` from JSON config is unrepresentable (`topology/bounded.rs:330-332`); `RELEASE_REQUIRED ≡ PERMITS.is_some()` — derivable const.

**B19. Layer leak + capability-dropping context.** (readers 3, 5) `register_resolved` takes `&ExpressionEngine` and walks JSON resolving `{{…}}` inside the resource crate (`manager/mod.rs:798-837`, acknowledged #718) — forces every caller/test to construct an ExpressionEngine. `ResourceContext::clone_for_acquire` silently rebuilds with **no-op accessors** (`context.rs:157-159`) — name advertises clone, performs capability downgrade; the no-op accessor then errors with `CoreError::CredentialNotConfigured` carrying a resource message (`context.rs:38-42`) while `try_acquire_any` returns `Ok(None)` (:54) — inconsistent failure semantics.

**B20. Events are strings, not data; no tenancy or correlation dimension.** (readers 0, 3) `AcquireFailed.error`, `RetryAttempt.error`, `SlotRefreshFailed.error`, `RecoveryGateChanged.state` pre-rendered `String` (`events.rs:43, 74, 87, 111`); events carry only `ResourceKey` — no scope/slot-identity (one tenant's failures indistinguishable) and no execution/workflow id (guard lifetime can't join a trace). Redaction "test" asserts on a hand-built literal (`events.rs:186, 197`) — tautological. `ResourceEvent::key() -> Option<&ResourceKey>` always Some (15/15 variants, `events.rs:136-154`).

**B21. Declared metrics never recorded; no per-resource dimension.** (readers 3, 5) `nebula_metrics::naming` declares `NEBULA_RESOURCE_ACQUIRE_WAIT_DURATION_SECONDS`, `USAGE_DURATION_SECONDS`, `CLEANUP_TOTAL`, `HEALTH_STATE`, `POOL_EXHAUSTED_TOTAL` (`crates/metrics/src/naming.rs:488-512`) — `ResourceOpsMetrics` records none (`metrics.rs:77-86`: 6 unlabeled global counters + 2 outcome-labeled rotation counters). No latency histograms, no pool gauges, no per-key labels. Pool-level asymmetry: pool never records acquire errors itself; Bounded records only the poison arm (`bounded.rs:351`); Unbounded handles are Owned so `record_release` never fires (`bounded.rs:365-367`). Violates the project's own observability-as-DoD rule.

**B22. Derive/macro hygiene cluster.** (readers 3, 5) Runtime `expect()` in emitted `key()`/slot registration instead of expansion-time `compile_error!` (`macros/src/resource.rs:74-77`, `field_slots.rs:230-235`); `purpose` attr parsed then `#[allow(dead_code)]` dropped (`field_slots.rs:36-38`); macro rustdoc contradicts the parser (example without `SlotCell` `macros/src/lib.rs:24-30` vs rejection at `field_slots.rs:150`; doc table lists `Option`/`Lazy` as accepted, parser rejects :167-177 — `optional`/`lazy` plumbing always `(true,false)`); `topology` attr informational only, unchecked against the actual `Pooled`/`Resident`/`Bounded` impl; diagnostics span the struct ident not the offending token (`resource_attrs.rs:63-92`, confirmed in `.stderr`); wrong-type attr value misdiagnosed as missing (`crates/sdk/macros-support/src/attrs.rs:61-66`). `ClassifyError`: discards the source chain (Display-string only, `macros/src/lib.rs:171`), `retry_after` compile-literal only (server Retry-After unrepresentable), non-rooted paths (`macros/src/lib.rs:166,193,196`), covers 5 of 8 ErrorKinds undocumented.

**B23. Phantom/misleading docs in load-bearing places.** (readers 1, 4, 0) Canon classifies README drift as a bug (§11.6): README's flagship example passes nonexistent field `resilience: None` (`crates/resource/README.md:253`, also :150) — does not compile; README says fan-out "lands in a follow-up" (:154, :185) but ROADMAP M11.5 marks it DONE (#688/#690/#703); `crates/credential-runtime/README.md:21` claims the M11.5 gap closed while `docs/ROADMAP.md:52-54, 921-924` says open (grep-verified: `register_resource_and_bind` zero callers) — ROADMAP is correct. ## D. STRATEGIC OPPORTUNITIES (max 5, researcher-grounded)

**D1. Credential lifecycle INSIDE the pool — the empty niche, half-built.** Research 0 ranks active auth lifecycle #4 of 10 and notes Temporal/Restate have *nothing*; research 1 shows the demand from below: deadpool #396 (AWS RDS IAM token refresh in-pool), sqlx `set_connect_options` born from IAM-token issues #445/#1500. nebula-resource already ships what no OSS Rust pool has: creation-time revoke-epoch fencing through idle/checkout/release/warmup/maintenance (`pool.rs:62-84, 715, 788-802, 1045-1055, 1210-1240`) and two-phase taint→drain→revoke with proof tokens. The play: finish M12.4 bind-population, add proactive pre-expiry refresh (ADR-0084's 1.1 item; `stale_after` is dead code today, B9), and market "rotate a credential, the pool converges with zero dropped requests" — no competitor in either the workflow or the Rust-pool market can say this.

**D2. Suspension-aware resource lifecycle for durable execution.** Trigger.dev v4 is the *only* mainstream design treating suspension as a resource event (`onWait`/`onResume` disconnect/reconnect around durable waits — research 0 §8) and it's a TS startup feature, not an engine primitive. Nebula owns both the durable engine and the resource manager: a guard-level `on_suspend`/`on_resume` (or engine-driven release-on-wait + reacquire-on-resume via the existing `ReleaseQueue`/acquire pipeline) would make nebula the first durable engine where a workflow parked for 3 days doesn't pin a connection. Directly composes with the Resident/Pooled topologies already built.

**D3. Durable, journal-attributable resource identity.** DBOS `DBOSConfiguredInstance` (recovery routes a resumed workflow to the *correct configured instance*) is the only durable-identity implementation in the field (research 0 §7); wasmtime's `ResourceTable` shows the mechanics — Copy/serializable typed id-handles with parent-child delete protection (research 1 §11). Canon already *demands* §13.3 attributability ("durable journal or operator-visible trace", `PRODUCT_CANON.md:433`) and today the seam is a drop-oldest in-memory EventBus (B20, gap G-§13.3 below) — i.e., the canon obligation and the market differentiator are the same feature: journaled acquire→use→release with id-handles that survive crash/recovery, fixing the aspirational orphan-drain claim (`lib.rs:30-32`) at the same time.

**D4. Compile-time resource binding — out-Dagster Dagster.** Dagster's entire 1.3 rework was escaping string indirection, and its post-rework complaint corpus is *still* about runtime gaps (research 0 §1); axum `State`/`FromRef` proves missing-state-as-compile-error is the DX gold standard (research 1 §10). Nebula has Rust: a typed registration funnel (kill B4's topology↔acquire redundancy), derive-driven action-side injection (`db: ResourceGuard<Postgres>` proven against registered rows at registration/activation, closing G7), and typed events/errors through the erased seam (B3) would deliver what no Python/TS orchestrator structurally can. The Bounded cap typestate shows the team already executes this pattern — point it at the seams that matter.

**D5. Tenant-true resource rows as a security story.** n8n's shared-credential account takeover (securityboulevard 2026-03, research 0 §6) and Windmill's path-only RBAC show the market's sharing models leak; research 0's cross-cutting lesson: secret-sharing semantics is where production holes appear. Nebula's `(key, scope, SlotIdentity::Structural)` row model with fail-closed `Ambiguous` is already structurally ahead — collision-free per-tenant pools with credential identity baked into row identity. To lead: per-row lifecycle (B6 fix), operator row enumeration (G-ops), and per-tenant quota/fairness inside shared rows, then publish it as "tenant isolation by construction, not by RBAC discipline."

## E. RISKS (3+ year horizon)

**E1. Hand-rolled pool maintenance is a permanent tax.** 2,121 LoC `runtime/pool.rs` + 613 LoC ReleaseQueue + 813 LoC RecoveryGate, all bespoke. The prior-art bug record is the warning: sqlx synchronized-expiry herd (#3051), acquire-loop starvation (#2848), maintenance task at 100% CPU (#3645); mobc's unrecoverable wedge; bb8 deadlocks (#122). nebula already carries 4 documented open races (#713–#716, `manager/mod.rs:209-235`) and **zero loom/model verification** of the load-bearing `(AtomicU64, Notify)` drain primitives and AcqRel TOCTOU close — despite the workspace's own `storage-loom-probe` precedent and the module doc calling the ordering safety-critical (`mod.rs:96-100`). The fork is justified *only* by the credential-fencing moat (reader 2's verdict); every table-stakes feature it lacks (C row "pooling") is interest accruing on that loan.

**E2. API freeze of accidental surface.** B9's dead-export inventory (whole Bounded topology, RecoveryGate family, RegisterOptions, Shared guards, 10-way method families, `ErasedAcquireFn`'s exported 5-level nested type `registry.rs:34-44`) plus a wholesale `pub use nebula_resource` from the SDK (`crates/sdk/src/lib.rs:51`) means 1.0 would lock in a surface ~5× larger than its real consumer set (two Manager entry points + rotation pair). Every future internal redesign then becomes a breaking change. Prefect's infra-blocks deprecation is the cautionary tale for overreached surface (research 0 §2).

**E3. Generics/typestate creep at an open-set boundary.** 4 assoc types where the pooled path proves `Lease = Runtime` ceremony (B15), `Capped<N>` making operator-configurable caps unrepresentable (B18), and `Bounded<Cap>` typestate at the plugin-facing trait — exactly what esp-idf-hal removed ("less flexibility and code duplication" across degraded variants) because resource kinds are an open set defined by third parties (research 1 §9). Meanwhile the same crate erases everything to `Box<dyn Any>` one layer up — paying both the generics tax and the erasure tax. In 3 years, every plugin author pays the first and every debugging session pays the second.

**E4. Discipline-encoded invariants become landmines under external authorship.** The crate's safety story repeatedly rests on prose: registration-funnel consistency over a pub `Registry::register` (B14), the registrar's residual pre-publish rotation race whose mitigation is "callers MUST quiesce" in a rustdoc comment (`registrar.rs:584-604`), topology attr unchecked against impls (B22), epoch folds hand-mirrored in hand-written impls (B5). Fine while one team writes every resource; structurally unsound the day plugins do. This directly violates the project's own type-enforce-over-discipline rule and is the highest-leverage design-phase target.

**E5. Doc mass as drift engine.** ~50% of the manager files are prose, the module doc doubles as ADR + changelog + bug ledger, and drift has already shipped in flagship spots: non-compiling README example (B23), inverted fingerprint doc (B1), phantom resilience pipeline (B23), fan-out status wrong in two READMEs. Under canon §11.6 every one of these is a bug. The pattern — canonical narrative restated 4–8× per concern — guarantees the lie rate grows with every refactor unless docs are consolidated to one authoritative block per invariant with pointers.

**E6. Observability debt compounds against the moat.** The product's differentiator is *operational* (credential lifecycle, tenant isolation) but the operational surface is the weakest layer: declared-but-unrecorded metrics (B21), string events with no tenant/execution dimension (B20), §13.3 attributability riding a drop-oldest ephemeral bus that canon §11.5 explicitly classifies as "never authoritative truth" (`PRODUCT_CANON.md:332, 433`), no operator row enumeration, no unbrick surface for PermanentlyFailed gates (B10). If unaddressed, the first serious production tenant incident is undiagnosable with the subsystem's own tooling — fatal for a product whose pitch is operational honesty.
