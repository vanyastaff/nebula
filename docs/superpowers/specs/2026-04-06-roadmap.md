# Nebula Public Roadmap

> Synthesized from 22 design specs, 15 conference rounds (220+ questions, 40+ companies), red team audit, and breakthrough ideas.

---

## Vision

Nebula is a modular, type-safe workflow automation engine in Rust. Single binary, embeddable as library or deployable as server. Local desktop, self-hosted, and SaaS from one codebase.

## Conference-Validated Decisions

These architectural choices were validated across 15 rounds, 40+ companies:

| Decision | Validated by | Status |
|----------|-------------|--------|
| serde_json::Value as universal type | Yuki (Rust), profiled OK | Confirmed |
| No DeploymentMode enum | n8n, Windmill, Supabase | Confirmed |
| Checkpoint-based, not event-sourcing | Temporal alumni, Restate | Confirmed |
| 5 action types (not more, not fewer) | All conference rounds | Confirmed |
| Credential = auth material only | 3 rounds of debate | Confirmed |
| One plugin per service (not marketplace) | WordPress, JetBrains, Shopify | Confirmed |
| ClickHouse for execution history | Honeycomb, ClickHouse architect | Confirmed |
| VictoriaMetrics over Prometheus | VictoriaMetrics founder | Confirmed |
| Tail-based sampling for traces | Jaeger maintainer | Confirmed |
| SandboxedContext Phase 1, WASM Phase 2 | Figma, Vercel, Fly.io | Confirmed |
| All plugins open source, Plugin Fund | User preference | Confirmed |
