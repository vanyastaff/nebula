# nebula-api

> **Note**: This crate is currently being established.

## Purpose
`nebula-api` serves as the API layer for Nebula, exposing endpoints for the control plane and other system components. It integrates with `nebula-engine` and `nebula-runtime` to provide external access to workflow management and execution.

## Dependencies (Planned)
- `axum`: Web framework
- `serde`: Serialization
- `nebula-core`: Core types
- `nebula-engine`: Workflow orchestration
