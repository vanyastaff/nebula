## Rules

### Error Enum Required Pattern
**Source**: `AGENTS.md` (Error Handling), `crates/action/src/error.rs`
**Rule**: When implementing a new error type in any library crate, the complete required pattern is:
1. `#[derive(Debug, thiserror::Error)]` — always both
2. `#[non_exhaustive]` — required for semver safety
3. Provide factory constructors accepting `impl Into<String>` for string-based variants (e.g., `pub fn execution_failed(msg: impl Into<String>) -> Self`)
4. Define `pub type <CrateName>Result<T> = Result<T, <CrateName>Error>` alias in the same module
5. Add `is_retryable()` / `is_fatal()` predicates where semantically meaningful
6. NEVER use `anyhow` — only `thiserror` in library crates

### New Crate lib.rs Template
**Source**: `AGENTS.md` (Module Structure), `crates/action/src/lib.rs`
**Rule**: When creating a new crate's `lib.rs`, always write it with this structure:
```
//! Crate-level doc comment.
//!
//! ## Quick Start
//! ...
//!
//! ## Core Types
//! ...

#![forbid(unsafe_code)]
#![warn(missing_docs)]

// ── Public modules ──────────────────────────────────────────────────────────
pub mod error;
pub mod prelude;

// ── Re-exports ───────────────────────────────────────────────────────────────
pub use error::MyError;

// ── Private modules ──────────────────────────────────────────────────────────
mod internal;
```
Every crate must expose `pub mod prelude` with star-importable common types. Use section dividers `// ── Section Name ────────────────────────────────────────────────────` between logical groups.

### Lint Suppression Style
**Source**: `AGENTS.md` (Lints section)
**Rule**: Never write `#[allow(lint)]`. The only accepted form is `#[expect(lint, reason = "<justification>")]`. Always write a concrete reason string. This is enforced by project convention and code review.

### Async Trait Pattern
**Source**: `AGENTS.md` (Traits & Async)
**Rule**: For async trait methods, use RPITIT by default — Rust edition 2024 supports `-> impl Future<Output = ...> + Send` natively without boxing overhead. Only use `#[async_trait::async_trait]` when the trait must be object-safe (`dyn Trait`). When using `async-trait`, add a brief comment explaining why object-safety is required. `async-trait` is a workspace dependency — no need to add it to individual crate `Cargo.toml` files.
