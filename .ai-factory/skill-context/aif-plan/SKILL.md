## Rules

### New Crate Workflow
**Source**: `AGENTS.md` (Adding a New Crate), root `Cargo.toml` (workspace members pattern)
**Rule**: When a plan involves adding a new crate, include these mandatory tasks:
1. `cargo new --lib crates/nebula-<name>` — crate names must use `kebab-case` with the `nebula-` prefix
2. Add `"crates/nebula-<name>"` to the `members` array in root `Cargo.toml`
3. In the new crate's `Cargo.toml`, use `workspace = true` for ALL `[package]` fields (version, edition, rust-version, authors, license, etc.) — never duplicate values
4. Create a proper `lib.rs` with `#![forbid(unsafe_code)]`, `#![warn(missing_docs)]`, and `pub mod prelude`
5. Before adding any cross-crate `[dependencies]`, apply the `nebula-layer-dependency-guard` mental model: does this import point downward in the layer order? If the signal can be published via `nebula-eventbus`, prefer that over a direct import.
