## Rules

### Rust Lint Attributes on New Crates
**Source**: `AGENTS.md` (Module Structure), `crates/action/src/lib.rs`, `crates/core/src/lib.rs`
**Rule**: Every new library crate `lib.rs` must open with `#![forbid(unsafe_code)]` and `#![warn(missing_docs)]`. Flag any PR that adds a new crate missing either of these attributes — their absence means unsafe code can silently enter the codebase and public APIs will pass CI without documentation.

### Lint Suppression Pattern
**Source**: `AGENTS.md` (Lints section)
**Rule**: Any `#[allow(lint)]` attribute in production code (outside `#[cfg(test)]` blocks) is a review finding. The only accepted form is `#[expect(lint, reason = "<explanation>")]`. Bare `#[allow]` is forbidden because it permanently silences the lint without justification. Flag every occurrence and request it be rewritten with a `reason`.

### Wildcard Imports, Error Type Completeness, and No anyhow
**Source**: `clippy.toml` (`warn-on-all-wildcard-imports = true`), `AGENTS.md` (Error Handling)
**Rule**: Flag all three of these patterns:
- (a) Wildcard imports (`use foo::*`) in production code — banned by `clippy.toml`'s `warn-on-all-wildcard-imports = true`
- (b) New `enum Error` types missing `#[non_exhaustive]` — required for semver safety so downstream crates don't break on new variants
- (c) `anyhow` dependency in any library crate — only `thiserror` is allowed in libs; `anyhow` is acceptable only in binary entry points and examples

### Layer Dependency Direction
**Source**: `AGENTS.md` (Project Structure), `CLAUDE.md` (ADR-003), `deny.toml`
**Rule**: Check cross-crate imports in the diff for layer direction violations. The permitted dependency direction is Infrastructure → Core → Cross-cutting → Business → Execution → API. Concrete rules:
- `nebula-core` may be imported by everything; it must not import from any other nebula crate
- Cross-cutting crates (`nebula-log`, `nebula-config`, `nebula-eventbus`, `nebula-metrics`, `nebula-telemetry`, `nebula-resilience`, `nebula-system`) may be imported at any layer
- Business crates (`nebula-credential`, `nebula-resource`, `nebula-action`, `nebula-plugin`) must not import execution or API crates
- Cross-business-layer signals must go through `nebula-eventbus` pub/sub, not direct imports
Flag any import that violates these rules — it will break `cargo deny check` in CI.
