## Rules

### Nebula Allowed Commit Types
**Source**: `commitlint.config.cjs` (project convention)
**Rule**: Nebula's commitlint defines `type-enum` with exactly these allowed types: `feat`, `fix`, `docs`, `style`, `refactor`, `perf`, `test`, `chore`. The types `build` and `ci` are NOT in the allowed list and will cause a commitlint error. Use `chore` for build system changes, dependency updates, tooling config, and CI pipeline edits. Never propose `build` or `ci` as commit types for this project.
