### Nebula CI Gate Sequence
**Source**: `.github/workflows/ci.yml`, `AGENTS.md` (CI Checklist section)
**Rule**: For Nebula, the complete CI checklist before any implementation is considered done is these 5 commands in order:
1. `cargo fmt --all -- --check`
2. `cargo clippy --workspace -- -D warnings`
3. `cargo check --workspace --all-targets`
4. `cargo test --workspace`
5. `cargo doc --no-deps --workspace`

All 5 must pass — a failure at any step is blocking. Clippy MUST be run with `-D warnings` (not the default lenient mode). The doc check is mandatory because `#![warn(missing_docs)]` is enforced and undocumented public items will cause CI to fail silently without it.
