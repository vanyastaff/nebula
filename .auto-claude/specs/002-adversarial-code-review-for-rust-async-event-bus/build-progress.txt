WHY NO CONCURRENCY IS POSSIBLE:

1. Event Bus Semantics:
   - Subscribers receive events in FIFO order
   - recv() must wait for next event in sequence
   - Parallelizing would violate ordering guarantees

2. Retry Loops:
   - Exponential backoff requires sequential iterations
   - Each sleep duration depends on previous iteration
   - Parallelizing makes no semantic sense

3. Stream Trait Constraint:
   pub trait Stream {
       fn poll_next(self: Pin<&mut Self>, ...) -> ...;
       //            ^^^^^^^^^ Mutable borrow prevents concurrent access

   - Tests emit then recv (emit must complete first)
   - Parallelizing would cause deadlocks

let mut sub2 = bus.subscribe();
